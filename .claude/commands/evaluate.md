# Evaluate - ブログ記事総合評価 v2.2

指定されたブログ記事を7つの観点から評価し、建設的なフィードバックを提供します。

## 使用方法

```
/evaluate [ファイルパス]
```

例:
```
/evaluate ../../blogs/blog.md
```

## 評価観点（7つ）

1. **防御力** (0.0-5.0) - 批判に耐えうるか
2. **思考整理力** (0.0-5.0) - 論理的に構造化されているか
3. **実践応用性** (0.0-5.0) - 読者が行動に移せるか
4. **構成と読みやすさ** (0.0-5.0) - 読者の理解を促進するか
5. **コミュニケーション力** (0.0-5.0) - 人間味があるか
6. **炎上リスク** (0.0-5.0) - 社会的配慮があるか（高いほど安全）
7. **人間らしさ** (0.0-5.0) - AIっぽくないか

---

## 実行内容

1. `tools/blog_evaluation/evaluation-prompt.md` を読み込み、指定されたファイルを評価
2. 評価レポートを**標準出力に表示**
3. 同じ内容を**ファイルに保存**:
   - 保存先: `tools/blog_evaluation/reports/`
   - ファイル名形式: `{記事ファイル名}-evaluation-{YYYY-MM-DD-HH-MM-SS}.md`
   - タイムスタンプは評価実行時刻（例: `2025-12-30-22-28-28`）

保存例:
```
tools/blog_evaluation/reports/mastodon-migration-evaluation-2025-12-30-22-28-28.md
```

---

## 評価の厳格性

- **5.0**: プロの編集者が見ても非の打ち所がない（稀）
- **4.0**: 商業メディアに掲載できる品質
- **3.0**: 一般的な技術ブログとして標準的
- **2.0以下**: 明確な改善が必要
- 迷った場合は低い方のスコアを付ける
